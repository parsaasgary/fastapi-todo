# UDEMY_fastapi

Simple FastAPI project built while following a FastAPI course. It demonstrates a small, practical API with user registration/login (JWT auth) and a TODO list resource.

Key ideas demonstrated:
- FastAPI routers and dependency injection
- JWT-based authentication and token verification
- Password hashing with `bcrypt`
- SQLAlchemy ORM setup and simple CRUD operations
- Pytest-based tests for endpoints and DB interactions

## Project structure
- `app/` – application package
- `app/Database/` – SQLAlchemy models and DB setup
- `app/router/` – FastAPI routers for `auth`, `user`, and `todos`
- `app/services/` – CRUD and service logic
- `app/utils/` – helpers for JWT, password hashing and test utilities
- `confg/credentials.env` – environment variables (not committed)
- `test/` – pytest test cases

## Requirements
Install runtime and test dependencies from `requirements.txt`:

```bash
python -m pip install -r requirements.txt
```

## Environment
Create `confg/credentials.env` with the following variables:

- `DATABASE_URL` (production DB)
- `DATABASE_TEST_URL` (test DB used by test suite)
- `SECRET_KEY` (JWT signing key)
- `ALGORITHM` (JWT algorithm, e.g. `HS256`)

Example `.env` (do NOT commit real secrets):

```
DATABASE_URL=sqlite:///./db.sqlite3
DATABASE_TEST_URL=sqlite:///./test_db.sqlite3
SECRET_KEY=your-secret-key
ALGORITHM=HS256
```

## Running the app
The project is a FastAPI app. You can run it with `uvicorn` (install separately if needed):

```bash
uvicorn app.main:app --reload
```

## Tests
Run tests with `pytest`:

## API Endpoints (summary)
- `GET /` — health check
- `POST /auth/add_user` — create a new user (payload: `CreateUserRequest`)
- `POST /auth/Token` — obtain access token (form: `username`, `password`)
- `GET /user/me` — get current authenticated user (requires `Authorization: Bearer <token>`)
- `POST /user/change-password/{user_id}` — change password for authenticated user
- `GET /todo/all_todos` — list todos for current user
- `GET /todo/todo/{todo_id}` — get a todo by id
- `POST /todo/add-todo` — create a todo for current user
- `PUT /todo/update-todo/{todo_id}` — update a todo
- `DELETE /todo/delete-todo/{todo_id}` — delete a todo

## Alembic (database migrations)
This project includes Alembic migration scripts under `app/alembic/` (generated revisions are in `app/alembic/versions/`). Alembic is configured via `app/alembic.ini` and `app/alembic/env.py`.

Notes about the configuration:
- `env.py` prepends the package path and imports `Database.database.Base` so Alembic can autogenerate migrations from the SQLAlchemy models.
- The `app/alembic.ini` file contains a `sqlalchemy.url` value; update it to match your `DATABASE_URL` or run Alembic with `-x`/environment overrides.

Common commands (run from the project root):

```bash
# install alembic if needed
pip install alembic

# create an autogenerated revision (edit message)
alembic -c app/alembic.ini revision --autogenerate -m "add new field"

# apply all migrations
alembic -c app/alembic.ini upgrade head

# downgrade one revision
alembic -c app/alembic.ini downgrade -1

# show current revision
alembic -c app/alembic.ini current

# stamp the DB without running migrations
alembic -c app/alembic.ini stamp head
```

If you use the `confg/credentials.env` for connection strings, consider editing `app/alembic/env.py` to load the same `.env` values (or set `sqlalchemy.url` in `app/alembic.ini`) so migrations target the intended database.

### Create a PostgreSQL database (example)
Below are example commands to create a PostgreSQL role and database you can add to `confg/credentials.env` as `DATABASE_URL` / `DATABASE_TEST_URL`.

Run these as a PostgreSQL superuser (e.g., `postgres`):

```bash
# create a role with password (replace values)
psql -U postgres -c "CREATE ROLE new_user WITH LOGIN PASSWORD 'newpassword';"

# create a database owned by the role
psql -U postgres -c "CREATE DATABASE fast_db OWNER new_user;"

# optionally create a test database
psql -U postgres -c "CREATE DATABASE fast_db_test OWNER new_user;"

# grant privileges (usually implicit for the owner)
psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE fast_db TO new_user;"
```

Then set the connection URL in `confg/credentials.env` (example):

```
DATABASE_URL=postgresql+psycopg2://new_user:Parsaasgary3@localhost:5432/fast_db
DATABASE_TEST_URL=postgresql+psycopg2://new_user:Parsaasgary3@localhost:5432/fast_db_test
```

Note: never commit real credentials. Use strong passwords and consider environment-specific secrets management for production.
